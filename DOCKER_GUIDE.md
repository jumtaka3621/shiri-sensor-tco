# Dockerでの実行ガイド

このガイドでは、`shiri_sensor_tco` アプリケーションをDockerを使って簡単に起動する方法を説明します。
この手順を行っても、普段の `npm run dev` での開発環境には一切影響ありません。

## 1. 準備（初回のみ）

まず、お使いのWindows PCに **Docker Desktop** がインストールされている必要があります。
まだの場合は、以下の公式サイトからインストールしてください。

- [Docker Desktop for Windows をインストール](https://docs.docker.com/desktop/install/windows-install/)
- インストール後、Docker Desktopを起動しておいてください。

---

## 2. アプリの起動

もっと簡単に起動できるように、専用のスクリプト（バッチファイル）を用意しました。

### 手順

1.  作業フォルダにある **`start_app.bat`** をダブルクリックしてください。
2.  自動的に黒い画面（ターミナル）が開き、アプリの構築・起動が始まります。
    ※ 初回は時間がかかります。
3.  起動が完了すると、自動的にブラウザが開いてアプリが表示されます。

---

## 3. アプリの停止

使い終わったら、**`stop_app.bat`** をダブルクリックしてください。
自動的にアプリが停止し、削除されます。

---

## よくある質問

Q. コードを修正したらどうすればいい？
A. Docker上のアプリに反映させるには、再度 `docker-compose up --build -d` を実行してください。再ビルドが行われます。
（普段の開発では、これまで通り `cd app` してから `npm run dev` を使う方が早くて便利です）

Q. ポート 8080 が使われているとエラーが出た
A. `docker-compose.yml` ファイルを開き、 `"8080:80"` の部分を `"8888:80"` など別の数字に変えて試してください。
